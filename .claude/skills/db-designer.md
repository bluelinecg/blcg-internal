---
name: db-designer
description: Design and validate database schema and query patterns.
---

Use this skill for any database-related changes.

---

## Responsibilities

- Schema design
- Relationship modeling
- Query structure validation
- RLS considerations

---

## Process

1. Define entities and relationships
2. Identify constraints and dependencies
3. Consider query patterns
4. Validate against data integrity rules

---

## Requirements

- All queries must live in /lib/db
- Must support dependency deletion rules
- Must respect RLS

---

## Output

- Schema proposal
- Query patterns
- Risks and tradeoffs

---

## Goal

Create a robust, scalable, and safe data model.
