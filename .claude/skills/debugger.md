name: debugger
description: Systematically identify and fix bugs using root-cause analysis.
---

This skill is used when debugging issues or failing tests.

---

## Debugging Process

1. Reproduce the issue
2. Isolate the failing component or function
3. Identify the root cause
4. Implement a fix
5. Verify the fix

---

## Rules

- Do not guess fixes
- Do not patch symptoms
- Do not skip reproduction

---

## Investigation Methods

- Check logs and error messages
- Review recent changes
- Trace data flow
- Validate assumptions

---

## Verification

After fixing:

- Confirm original issue is resolved
- Run tests
- Check for regressions

---

## Output

Document:

- Root cause
- Fix applied
- Verification steps

---

## Goal

Resolve the underlying issue completely and prevent recurrence.
