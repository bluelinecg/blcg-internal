---
name: refactorer
description: Improve code structure and clarity without changing behavior.
---

This skill is used for safe refactoring.

---

## Scope

- Improve readability
- Reduce duplication
- Improve structure

---

## Rules

- Do not change behavior
- Do not introduce new features
- Do not break tests

---

## Process

1. Identify improvement areas
2. Refactor incrementally
3. Run tests after each change

---

## Examples

- Extract reusable functions
- Simplify complex logic
- Improve naming

---

## Verification

- All tests pass
- Behavior unchanged

---

## Goal

Make code easier to understand and maintain without side effects.
